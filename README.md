# GovernmentProposal

정부 제안서 작성 및 관리를 위한 모듈화 레포지토리

---

## 목적

본 레포지토리는 정부 지원사업 제안서를 체계적으로 작성·관리하기 위한 공간임.
각 프로젝트별 디렉토리를 독립적으로 구성하고, 제출서류와 참고서류를 분리하여 관리함.

---

## 레포지토리 구조

```
GovernmentProposal/
├── README.md                        # 본 파일
├── CLAUDE.md                        # AI 작성 가이드 (문체 규칙, 스타일 가이드)
└── Projects/
    └── [연도-번호]프로젝트명/
        ├── document_structure.md    # 제출 문서 목차 및 작성 구조 (수정 금지)
        ├── CLAUDE.md                # 프로젝트별 AI 작성 지침 및 참조 데이터
        ├── Main_Documents/          # 실제 제출서류
        └── Ref_Documents/           # 참고서류 (공고문, 논문, 기사 등)
```

### 프로젝트 명명 규칙

```
[YYYY-MM]프로젝트_식별명
예: [2026-03]opensource_AISW
```

---

## 새 프로젝트 추가 방법

1. `Projects/` 디렉토리 하위에 `[YYYY-MM]프로젝트명` 형식으로 디렉토리 생성
2. `document_structure.md` 파일을 생성하고 제출 문서의 목차 구조 정의
3. `Main_Documents/` 와 `Ref_Documents/` 디렉토리 생성
4. 공고문, 안내서, 평가항목 등 참고서류를 `Ref_Documents/` 에 배치
5. 프로젝트별 `CLAUDE.md` 작성 (참조 데이터 경로 및 작성 지침 명시)

---

## 문서 작성 규칙

### 필수 문체: 음슴체

정부 제안서 전 문서는 반드시 음슴체를 사용함.

```
올바른 예시
"메모리 모듈을 통합하여 성능을 향상시킴"
"평가 체계를 수립함"

금지 예시
"메모리 모듈을 통합하여 성능을 향상시킵니다"
"평가 체계를 수립합니다"
```

### 이모지 사용 금지

제출 문서(Main_Documents/) 내에서는 이모지를 절대 사용하지 않음.
가이드 문서(CLAUDE.md, README.md)는 예외적으로 허용.

### 문장 패턴

```
[문제/배경] + [해결책] + [기대 결과]
```

### 수치 표현

- 구체적 수치 필수: "20% 이상", "150B급", "64K 토큰"
- 모호한 표현 금지: "많은", "빠른", "좋은"

---

## document_structure.md 준수 의무

`document_structure.md` 에 정의된 문서 구조는 절대 변경하지 않음.
제출서류의 섹션 순서, 항목, 목차는 해당 파일을 기준으로 작성함.

---

## Claude Code 사용 시 유의사항

본 레포지토리는 Claude Code(AI 코딩 어시스턴트)와 함께 사용하도록 구성되어 있음.

- 루트 `CLAUDE.md`: 전체 공통 문체 및 스타일 규칙 정의
- 프로젝트별 `CLAUDE.md`: 해당 프로젝트의 참조 데이터 경로 및 추가 작성 지침 정의
- AI가 문서 작성 시, 반드시 해당 프로젝트의 `document_structure.md` 를 기반으로 작성함
- 작성 요령 파트는 실제 내용 작성 후 제거함
